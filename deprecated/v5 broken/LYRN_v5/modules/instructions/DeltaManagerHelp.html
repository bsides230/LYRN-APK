What is the Delta Manager?

The Delta Manager is a system module that allows you to dynamically inject real-time data or state changes ("deltas") into the core system without requiring a full restart or permanent modifications to base files.

These deltas act as a layer on top of the system's baseline memory and context, allowing for dynamic overrides, live statistics feeds, or sensor data to be incorporated into the LLM's active context continuously.

Key Concepts

  • Data Streams: Ongoing updates sent to the Delta Manager representing live data (e.g., current time, system stats).
  • Scheduled Scripts: Python scripts located in the deltas/ directory that are automatically polled by the system at a set interval to feed new data.
  • Manifest: The internal tracking file (deltas/_manifest.json) that stores the current state of all enabled streams.

How to Use the Delta Manager

Managing Data Streams

The Active Data Streams section displays all streams currently being tracked by the system. You can:

  • Enable/Disable: Use the toggle switch to temporarily hide a stream from the LLM's context without deleting it.
  • Delete: Remove the stream entirely from the manifest using the trash icon.
  • Manual Entry: Click "Add Delta" to manually create a static stream for testing or one-off overrides.

Managing Scheduled Scripts

The Scheduled Scripts section automatically discovers any .py files placed inside the root deltas/ directory.

  • Enable Polling: Toggle the switch to have the system automatically execute the script in the background.
  • Interval: Set how frequently (in seconds) the script should be run. Be mindful of performance—frequent polling of heavy scripts may impact system resources.
  • Last Run: Displays the timestamp of the last successful execution.

Extending with Scripts

You can create custom Python scripts to feed data into the Delta Manager. Place these scripts in the deltas/ directory at the root of the project.

The scripts simply need to send a POST request with JSON data to the /api/deltas/update endpoint.

Sample Script (send_timestamp.py)

Below is a minimal, dependency-free template you can use to start building your own delta scripts:

import urllib.request
import urllib.error
import json
import time

# Configuration
CORE_URL = "http://localhost:8000"
DELTA_NAME = "System Time"

def get_current_time():
    """Generates the data to be sent."""
    # This could be anything: reading a sensor,
    # checking an API, getting OS stats, etc.
    return time.strftime("%Y-%m-%d %H:%M:%S")

def send_delta(name, value):
    """Sends the data to the Delta Manager."""
    url = f"{CORE_URL}/api/deltas/update"
    data = json.dumps({
        "name": name,
        "value": value
    }).encode('utf-8')

    req = urllib.request.Request(url, data=data, headers={
        'Content-Type': 'application/json'
        # Add 'X-Token': 'your_token' if authentication is required
    })

    try:
        urllib.request.urlopen(req)
        print(f"Successfully sent delta: {name} = {value}")
    except urllib.error.URLError as e:
        print(f"Failed to send delta: {e}")

if __name__ == "__main__":
    current_time = get_current_time()
    send_delta(DELTA_NAME, current_time)

Note: Scripts run by the scheduler should execute once and exit. The system handles the repeated execution based on the polling interval you configure in the GUI.
