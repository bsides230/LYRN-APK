# Delta Key Reference

This document serves as a central reference for all "delta keys" used within the LYRN-AI system. A delta represents a single, atomic change to the AI's state or configuration. Each system component that generates deltas must have a unique key assigned to it from this file.

## Purpose

- **Traceability**: Provides a clear way to trace the origin of any change in the system.
- **Debugging**: Helps in debugging by identifying which component is responsible for a specific state change.
- **Standardization**: Ensures a consistent method for logging and managing state modifications across different parts of the application.

## Key-Value Structure

The `key` from this file should be included in the delta's data. This requires a `key` field in the delta object structure.

## Delta Keys

| Scope         | Target | Key   | Description                                                                                                                        |
|---------------|--------|-------|------------------------------------------------------------------------------------------------------------------------------------|
| `personality` | `traits` | `P-001` | Represents updates to the AI's personality traits, typically originating from the Personality Sliders in the GUI.                    |
| `heartbeat`   | `*`    | `H-001` | Represents internal state updates generated by the AI's heartbeat reflection process. The target can vary depending on what the AI decides to modify. |

*This document should be updated whenever a new delta-generating mechanism is added to the system.*
